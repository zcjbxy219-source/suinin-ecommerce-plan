3技术规格详解

3.1 Mesh 建模规范

参数规范值说明
三角面上限（配件）4,000 tri超过自动被审核拒绝，建议预留 10% 余量 → 建模目标 ≤3,600
三角面上限（Layered Clothing）10,000 tri衣服类可以更复杂，但仍需优化
UV 通道单一通道 [0,1]禁止 UV 超出 0~1 范围，禁止重叠 UV 岛
UV 岛间距≥2 px防止压缩 mip 时纹理溢色（bleeding）
变换矩阵全部 Apply导出前 Ctrl+A → Apply Scale/Rotation，scale 必须为 1
法线方向朝外（ outward ）背面剔除模式，法线反了会显示黑色
零面积面不允许清理所有退化几何（degenerate geometry）
非流形边不允许使用 Blender 的 Select → Non-Manifold 检查
导出格式.fbx（有骨骼）/ .obj（静态）FBX 用于 Layered Clothing 和需要变形的配件

3.2 纹理贴图规范

参数要求原因
分辨率范围256×256 ~ 1024×1024超过 1024 自动缩放，细节丢失
格式PNG（RGBA）支持 Alpha 透明通道
色深8-bit/channel标准 RGBA8888
版权内容❌ 绝对禁止任何 Logo、品牌名、现实 IP → 审核拒绝
文字内容⚠️ 极度谨慎中文需审核，推荐用抽象图形代替文字
不适当内容❌ 禁止暴力、武器暗示、成人内容一律拒审
✅ 纹理实操建议：拼图发夹上的「碎」字可以用抽象笔画或符号代替真实汉字，降低审核风险。明信片的「邮戳」用圆形几何图案+心形即可，不写真实地名。围巾上的手写纹理做成类似手写但不可读的抽象线条。

3.4 Layered Clothing 特殊要求（T2 衣服类）

Roblox 的 Layered Clothing 系统比传统 Shirt/Pants 更复杂，需要额外准备以下文件：

  • [ItemName].fbx — 外层可见网格（有 UV 和纹理）
  • [ItemName]_InnerCage.fbx — 内收缩网格（比外层小 ~0.01 studs），定义衣物如何贴合身体
  • [ItemName]_OuterCage.fbx — 外扩展网格（比外层大 ~0.01 studs），允许其他叠穿物品堆叠在其上
  • 骨骼权重 — 所有顶点必须加权到正确的 R15 骨骼，无未加权顶点（否则撕裂）
  • 测试体型 — 必须在 Young / Classic / Normal / Rthro Narrow / Rthro Broad 五种体型下测试
ℹ️ 内笼（InnerCage）的重要性：缺少 InnerCage 会导致衣物穿模——直接穿透进角色身体里。这是新手最容易踩的坑，也是 Roblox 审核时的高频退回原因。

4DCC → Roblox Studio 制作流程

Phase 1：建模（Blender）

  1. 导入 Roblox 官方 R15 参考骨骼（从 Roblox 官网下载 R15 Reference Rig）
  2. 以参考骨骼为基准建立配件网格，初始建模即控制三角面在限制内
  3. UV 展开，确保单通道 [0,1]，岛间距 ≥2px
  4. 对 Layered Clothing 项目：复制外层网格 → 缩放生成 InnerCage（Scale 0.98）和 OuterCage（Scale 1.02）
  5. 权重绘制：使用 Transfer Weights 从参考骨骼转移到新网格
  6. 检查零面积面和非流形边（3D Viewport → Overlay → Geometry All → 开启 Face Orientation）
  7. Apply All Transforms（Ctrl+A → All Transforms）

Phase 2：纹理绘制

  1. 导出 UV 布局图（UV Editor → UVs → Export UV Layout）
  2. 在 Photoshop / Procreate / Krita 中按 UV 布局绘制纹理
  3. 遵循品牌色彩规范（马卡龙六色系）
  4. 保存为 PNG（RGBA），分辨率不超过 1024×1024
  5. 检查 Alpha 通道：边缘平滑过渡，无硬锯齿

Phase 3：Studio 导入与测试

  1. 打开 Roblox Studio → Avatar → Import Accessory（配件）或 Import Layered Clothing（衣服）
  2. 导入 .fbx 文件和对应 .png 纹理
  3. 在 Avatar Editor 中预览效果
  4. 五个体型测试：依次切换 Classic / Young / Normal / Rthro Narrow / Rthro Broad
  5. 五组动作测试：idle（站立）、walk（走路）、run（跑步）、jump（跳跃）、sit（坐下）
  6. 逐帧检查穿模情况（clipping），记录问题帧
  7. 回到 Blender 修正 → 重新导入 → 再次测试

Phase 4：提交准备

  1. 准备 Icon 缩略图：420×420 PNG，纯色背景，清晰展示商品
  2. 填写元数据：名称准确描述、分类正确、价格合理
  3. 撰写描述文案：突出品牌特色和使用场景
  4. 最终自查清单执行（见 Part 5）
  5. 通过 Creator Dashboard 提交

5提交前最终自查清单

每一款配件提交前，逐一核对以下项目：

🧱 Mesh 检查

  • 三角面数 ≤ 4,000（配件）/ ≤ 10,000（Layered Clothing）
  • 单一网格对象（Single Mesh Object）
  • 单一 UV 通道，所有 UV 在 [0,1] 范围内
  • 无重叠 UV 岛
  • All Transforms Applied（scale = 1, rotation = 0）
  • 枢轴点（Pivot Point）位于正确的 Attachment 位置
  • 无零面积面、无非流形边

🎨 纹理检查

  • 分辨率在 256~1024 px 之间
  • PNG 格式，RGBA 色彩空间
  • UV 岛间距 ≥ 2px
  • 无任何版权素材、品牌 Logo、真实 IP 元素
  • Alpha 通道正确（透明部分确实透明）

🔗 挂载与兼容性检查

  • Attachment 对象存在且命名正确（如 HatAttachment）
  • 已在 5 种 R15 体型下测试通过
  • 在 idle / walk / run / jump / sit 动作中无穿模
  • Layered Clothing 类包含 InnerCage 和 OuterCage
  • 所有顶点已加权，无遗漏

📦 文件与元数据检查

  • 文件格式正确（FBX/OBJ）
  • 文件命名规范：[CreatorName]_[ItemName]_[Type]
  • Icon 缩略图 420×420 PNG，清晰展示商品全貌
  • 商品名称准确描述，不含误导信息
  • 价格参考同类竞品，偏差 ≤15%

6.2 玩家装扮系统 — HumanoidDescription 应用

-- ServerScriptService/AvatarSystem.lua
local Players = game:GetService("Players")
local DataStoreService = game:GetService("DataStoreService")
local outfitStore = DataStoreService:GetDataStore("PlayerOutfits")

local AvatarSystem = {}

-- 碎碎念 UGC 配件 Asset ID 表（提交后填入实际值）
local SUI_ASSETS = {
    hairClip     = "",   -- SUI-001
    headband     = "",   -- SUI-002
    earrings     = "",   -- SUI-003
    sweater      = "",   -- SUI-004
    skirt        = "",   -- SUI-005
    scarf        = "",   -- SUI-006
    backpack     = "",   -- SUI-007
    handbag      = "",   -- SUI-008
    wings        = "",   -- SUI-009
}

-- 一键穿上「念念同款」全套
function AvatarSystem.applyFullSet(player: Player)
    local char = player.Character
    if not char then return end
    
    local humanoid = char:FindFirstChildOfClass("Humanoid")
    if not humanoid then return end
    
    local desc = humanoid:GetAppliedDescription()
    
    desc.HatAccessory   = SUI_ASSETS.hairClip
    desc.HairAccessory  = SUI_ASSETS.headband
    desc.FaceAccessory = SUI_ASSETS.earrings
    desc.Shirt         = SUI_ASSETS.sweater
    desc.Pants         = SUI_ASSETS.skirt
    
    
    humanoid:ApplyDescription(desc)
end

-- 保存玩家自选搭配
function AvatarSystem.saveOutfit(player: Player, outfitData: table)
    local success, err = pcall(function()
        outfitStore:SetAsync(player.UserId .. "_outfit", outfitData)
    end)
    return success
end

-- 加载玩家保存的搭配并在重生时应用
function AvatarSystem.loadAndApply(player: Player)
    local success, data = pcall(function()
        return outfitStore:GetAsync(player.UserId .. "_outfit")
    end)
    
    if success and data then
        local char = player.Character
        local humanoid = char and char:FindFirstChildOfClass("Humanoid")
        if humanoid then
            local desc = humanoid:GetAppliedDescription()
            for key, assetId in pairs(data) do
                desc[key] = tostring(assetId)
            end
            humanoid:ApplyDescription(desc)
        end
    end
end

return AvatarSystem

Roblox UGC 不是独立的收入来源，而是品牌延伸和获客渠道：

  • UGC 商品描述中加入「来自碎碎念情绪疗愈拼图品牌」的品牌声明
  • Icon 缩略图使用与实体包装一致的视觉风格，强化品牌认知
